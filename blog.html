Q1. What are the deferences between var, let and const?

The scope of a var variable is a functional scope and the scope of a let, const variable is a block scope. Var can be updated and re-decleared into the scope. let and const can be updated but can not be re-decleared into the scope. var can be decleared without initialization but let and const can not be decleared without initialization.

Q2. Deferences between Arrow function and Normal function.

Arrow function allows a develper to get the same result like normal function but in a few lines of code and approximately half the typing. Unlike normal function arrow function do not have their own this. The value of this inside an arrow function remains the same throughout their lifecycle of the function. Arrow function can never have duplicate name parameters.

Q3. What are differences of map(), filter(), foreach() and find()?

.forEach() is used to execute the same code on every element in an array but does not change the array and it returns undefined.
.map() executes the same code on every element in an array and returns a new array with the updated elements.
.filter() checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria.
.find() method returns the first element in the provided array that satisfies the provided testing function. If no values satisfy the testing function, undefined is returned.

Q4. Why would i use template string?

Template literals are sometimes informally called template strings. Template literals use back-ticks(``) rather than the quotes("") to define a string. With template literals, i can use both single and double quotes inside a string. Template literals provide an easy way to interpolate variables and expressions into strings.
